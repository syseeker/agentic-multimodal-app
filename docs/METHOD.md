# The Method — building an agentic app with NVIDIA skills

This is the **developer persona** walkthrough. The point of this repo is not just the running app — it's the *reproducible method* your team can apply to any LLM application: drive a coding agent (Claude Code) augmented with **NVIDIA skills** to assemble an agentic system from NVIDIA building blocks.

## 1. Install the NVIDIA skills into your coding agent

Skills are installed into **your own** Claude Code — not vendored into this repo:

```bash
npx skills add nvidia/skills --skill aiq-research        --agent claude-code
npx skills add nvidia/skills --skill nemo-retriever      --agent claude-code
npx skills add nvidia/skills --skill rag-blueprint       --agent claude-code
npx skills add nvidia/skills --skill nemotron-speech     --agent claude-code
npx skills add nvidia/skills --skill vss-ask-video       --agent claude-code
npx skills add nvidia/skills --skill nemotron-policy-generator --agent claude-code
npx skills add nvidia/skills --skill data-designer       --agent claude-code
```

Verify any skill's signature:
```bash
pip install model-signing
model_signing verify certificate <skill_dir> --signature <skill_dir>/skill.oms.sig \
  --certificate_chain nv-agent-root-cert.pem --ignore_unsigned_files
```

## 2. Which skill maps to which part

> **Honest note:** the code in this reference repo was authored directly as a
> worked example — it was **not** generated by invoking these skills one-by-one.
> The table below is the *map a developer follows* to (re)generate or extend each
> part with the skill-augmented method. Treat it as the recipe, not a build log.

| Part of this repo | Skill to use | NVIDIA component |
|---|---|---|
| `app/engines/deepagents_engine.py` | `aiq-research` | AI-Q deep-research pattern |
| `app/tools/audio_extract.py` | `nemotron-speech` | Riva / Parakeet ASR, MERaLiON serving |
| `app/tools/image_extract.py` | `vss-ask-video` | VSS / VLM Q&A |
| `app/tools/graph_rag.py` | `nemo-retriever`, `rag-blueprint` | NeMo Retriever + Milvus/cuVS |
| `app/guardrails/` | `nemotron-policy-generator` | NeMo Guardrails policies |
| `data/` | `data-designer` | NeMo Data Designer synthetic data |
| `observability/`, `benchmark/` | NeMo Agent Toolkit docs | `nvidia-nat`, aiperf, Nsight |

## 3. The loop

For each building block:
1. **Ask the skill-augmented agent** to scaffold the integration ("wire NeMo Retriever over Milvus for case-file RAG").
2. **Review** the generated code against the skill's reference patterns.
3. **Run** the verification step (see each PR's section in the repo plan).
4. **Commit** with a note citing the skill used.

The result is this repo: a worked example you can fork and retarget by editing `app/config/config_socrates.yml`.
